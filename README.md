A state-of-the-art, premium Human Resource Operating System (HROS) built to manage attendance, employees, tasks, meetings, and organizational announcements. Features an interactive, responsive layout, rich statistics, real-time charts, and role-based access.

---

## ✨ Features

*   📅 **Attendance Tracking**: Clock in/out, check history, and monitor logs.
*   👥 **Employee Directory**: Manage staff records, roles, salaries, and positions.
*   📝 **Task Manager**: Create, assign, prioritize, and track task progress.
*   🤝 **Meeting Scheduler**: Schedule meetings, set locations, link Google Meet, and manage invitees.
*   📣 **Announcements Bulletin**: Post company announcements with pinning and priority levels.
*   📊 **Rich Analytics**: Visualized attendance trends, department breakdowns, and task metrics.

## 📂 Project Architecture & Monorepo Structure

This project is configured as a **pnpm Monorepo** containing modular workspaces:

```text
hros/
├── artifacts/
│   ├── api-server/         # Express.js API backend (PostgreSQL + Drizzle)
│   ├── hr-dashboard/       # React + Vite frontend dashboard (shadcn/ui + Tailwind v4)
│   └── mockup-sandbox/     # Isolated UI preview sandbox environment
├── lib/
│   ├── db/                 # Database schema declaration, configuration, and migrations
│   ├── api-spec/           # OpenAPI 3.0 specification & Client Generators (Orval)
│   ├── api-zod/            # Shared Zod validation schemas
│   └── api-client-react/   # Autogenerated API React Query hooks
├── scripts/                # Development utility & automation scripts
└── package.json            # Monorepo configuration
```

## 🛠️ Getting Started

### 📋 Prerequisites
- **Node.js**: `v20.x` or later
- **pnpm**: `v9.x` or later
- **PostgreSQL**: Local or cloud-hosted instance running

### 1. Install Dependencies
Install all monorepo dependencies cleanly at the root:
```bash
pnpm install
```

### 2. Configure Database Environment Variable
Define the connection string for your PostgreSQL database. Create a `.env` file or export it:
```bash
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/postgres
```

### 3. Push Database Schema
Apply the database schemas directly via Drizzle Kit:
```bash
pnpm --filter @workspace/db run push
```

### 4. Running Development Servers

Start the backend and frontend services side-by-side:

#### 🖥️ Backend API Server (Port `8080`)
```powershell
$env:PORT="8080"
$env:DATABASE_URL="postgresql://postgres:postgres@localhost:5432/postgres"
pnpm --filter @workspace/api-server run dev
```

#### 🎨 Frontend Dashboard (Port `5173`)
```powershell
$env:PORT="5173"
$env:BASE_PATH="/"
pnpm --filter @workspace/hr-dashboard run dev
```

The frontend dashboard will automatically proxy API calls (`/api`) to the backend server.
Access the web app at **[http://localhost:5173](http://localhost:5173)**.
